# Module launch points

Run commands from the project root:

```powershell
python .\1_row_ECG_feature_extractor\run.py
python .\2_ECG_feature_derived\run.py
python .\2_context_features\run.py
python .\3_dataset_builder\run.py
python '.\4_feature_analyses (устарел)\run.py'
```

Install runtime dependencies first:

```powershell
python -m pip install -r requirements.txt
```

Each module has the same simple convention:

- `input/` contains module input files.
- `output/` receives module output files.
- `../protocol.yaml` is the main editable protocol.
- `output/_runtime_protocol.yaml` is generated by `run.py` when a module needs absolute input/output paths.

Current paths:

- `1_row_ECG_feature_extractor/run.py`
  - input: `1_row_ECG_feature_extractor/input/sample_data`
  - output: `1_row_ECG_feature_extractor/output/features_protocol.csv`
  - reviews: `1_row_ECG_feature_extractor/output/_reviews`
- `2_ECG_feature_derived/run.py`

  - output: `2_ECG_feature_derived/output/features_derived.csv`

  - output: `2_context_features/output/context_computed.csv`

- `3_dataset_builder/run.py`

The old launch files are kept only for compatibility:

- `1_row_ECG_feature_extractor/run_extract.py`
- `3_dataset_builder/run_dataset_build.py`
